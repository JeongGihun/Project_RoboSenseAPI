# /write — 4~10단계 (GREEN + REFACTOR + QA + 학습 + 커밋 + 검증 + 머지)

.claude/tasks/todo.md에서 현재 진행 중인 작업 확인.
todo.md에 진행 중 작업이 없으면 "/plan 먼저 실행하세요" 안내.

## 4단계: 구현 (TDD GREEN)
- 3단계에서 작성한 실패 테스트를 통과하는 최소 코드 작성
- pytest 실행해서 GREEN(통과) 확인
- 실패 시: 에러 분석 → 자동 수정 → 재실행 (3회까지, 이후 사용자에게 보고)

## 5단계: 리팩토링 (TDD REFACTOR)
- 중복 제거, 구조 개선
- pytest 재실행해서 GREEN 유지 확인
- REFACTOR 후 테스트 깨지면 자동 롤백 후 재시도

## 6단계: QA
- pytest 전체 실행 (기존 테스트 포함)
- 커버리지 확인 (70%+ 목표)
- 실패하는 테스트 있으면: 원인 분석 → 수정 → 재실행
- 전체 통과될 때까지 반복

## 7단계: 학습 기록
- 작업 중 잘못 판단했거나 배운 것이 있으면 .claude/tasks/lessons.md에 추가 제안
- 사용자 승인 후 저장
- 없으면 스킵

## 8단계: 커밋
- feature 브랜치에 커밋
- 변경 파일 확인 후 스테이징
- 커밋 메시지: 한글 OK, 간결하게

## 9단계: 로컬 검증
- Docker 재빌드: docker-compose build && docker-compose up -d
- /health 스모크 테스트: curl localhost/health
- 실패 시: 로그 확인 → 원인 분석 → 수정 → 6단계부터 재실행

## 10단계: integration 머지
- feat/* → integration PR 생성 (gh pr create)
- CI 통과 확인
- CI 실패 시: 로그 확인 → 수정 → 6단계부터 재실행

## 에러 처리 원칙
- 각 단계에서 실패 시 자동으로 원인 분석 → 수정 → 해당 단계 재실행
- 3회 연속 실패 시 사용자에게 상황 보고 후 판단 요청
- 서브에이전트 활용해서 에러 원인 탐색, 메인 에이전트(나)가 검토 후 수정 결정

## 완료 후
- todo.md 해당 작업 체크박스 완료 처리
- "작업 완료" 요약 출력 (완료 항목, 테스트 수, 커버리지)
